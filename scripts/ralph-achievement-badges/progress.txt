## Codebase Patterns
- Shared types are all in a single `packages/shared/src/index.ts` file ‚Äî add new types at the bottom
- Use `/Users/neil.feris/.nvm/versions/node/v22.2.0/bin/npx tsc --project <path>/tsconfig.json --noEmit` for typecheck (shell has broken nvm lazy-loading, call node binary directly)
- lint-staged runs on commit: eslint --fix + prettier --write
- XP types already exist (XP_AWARDS, calculateLevel, etc.) ‚Äî achievement XP rewards should integrate with existing awardCustomXP flow
- Existing pattern for criteria-style constants: use `as const` arrays and Record types
- Startup migrations: version-tracked in `_startup_migrations` table, next available version is 7
- Seeding pattern: use `INSERT ... ON CONFLICT(code) DO UPDATE SET ...` for upsert logic in SQLite
- Schema index pattern: `(table) => [uniqueIndex('name').on(table.col1, table.col2)]`
- All user-scoped tables: `userId` references `users.id` with `ON DELETE CASCADE`
- `awardCustomXP(userId, amount, source)` uses source as idempotency key ‚Äî for achievements use `achievement:{code}` as source
- Service pattern: export individual functions, import db/schema from `../db/index.js`, import shared types from `@ace-prep/shared`
- Don't import unused drizzle operators ‚Äî lint-staged will fail the commit
- lint-staged checks entire file, not just your changes ‚Äî fix any pre-existing unused imports in files you touch
- Route auth pattern: `fastify.addHook('preHandler', authenticate)` at top of route function
- Routes registered in `packages/server/src/index.ts` with `fastify.register(routes, { prefix: '/api/...' })`
- `userStreaks.currentStreak` gives live streak count ‚Äî no date computation needed
- SR reviews are handled in `packages/server/src/routes/questions.ts` (POST /review endpoint)
- Both study.ts and drills.ts had unused `{ db, schema }` ‚Äî only `db` is used; check all route files
- `isNotNull(spacedRepetition.lastReviewedAt)` gives count of reviewed SR cards for a user
- `streakResult.streak.currentStreak` is the post-update streak value ‚Äî use this when passing streak to achievement context
- Learning path completion (POST `/learning-path/:certId/:order/complete`) in study.ts ‚Äî needs both streak and completionist achievement checks
- `performanceStats` table is NOT actively written by routes ‚Äî domain-level stats are computed from `examResponses`/`studySessionResponses` via SQL GROUP BY on `domains.id`
- Client components: CSS Modules + CSS variables from globals.css, export from `components/common/index.ts`
- Rarity color mapping: common=#4a9eff (blue), rare=#a855f7 (purple), epic=#f59e0b (gold)
- Card hover pattern: `translateY(-2px)` lift + `var(--shadow-md)` + reveal tooltip via opacity
- Fonts: Outfit for text, JetBrains Mono for data/counts/labels ‚Äî set via `var(--font-sans)` and `var(--font-mono)`
- Page route pattern: `<ProtectedRoute><AppShell><RouteErrorBoundary><Component /></RouteErrorBoundary></AppShell></ProtectedRoute>`
- Client API pattern: define response interfaces, then `export const fooApi = { method: () => request<Type>('/path') }`
- React Query invalidation: hierarchical keys like `['achievements']` and `['achievements', 'progress']` allow targeted cache busting
- Notification utility pattern: create a util in `utils/` that imports `showToast` + `queryClient` directly for use outside React components (see `streakNotifications.ts`, `achievementNotifications.ts`)
- Shared package rebuild required: after changing shared types, run `npm run build -w @ace-prep/shared` before client typecheck picks up changes
